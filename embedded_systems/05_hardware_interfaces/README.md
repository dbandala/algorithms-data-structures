# 05 — Hardware Interfaces

Hardware interfaces connect the microcontroller to sensors, actuators, and other
ICs. Senior embedded engineers are expected to understand protocol details at the
bit and signal level — not just the API.

---

## Key Concepts

### GPIO (General Purpose Input/Output)
The simplest peripheral. Configuration registers control:
- **Direction**: input or output
- **Output type**: push-pull (drives high AND low) vs open-drain (only drives low)
- **Pull resistors**: pull-up (high when floating), pull-down (low when floating)
- **Speed**: slew rate — fast edges cause EMI; use the slowest acceptable speed
- **Alternate function**: route pin to UART/SPI/I2C peripheral instead of GPIO

### UART (Universal Asynchronous Receiver/Transmitter)
Point-to-point, no clock line — both sides must agree on baud rate.

**Frame format**: `[START | D0 D1 D2 D3 D4 D5 D6 D7 | PARITY | STOP]`
- Start bit: always 0 (space)
- Stop bit: always 1 (mark) — 1 or 2 stop bits
- Parity: even, odd, or none

**Baud rate error**: actual baud = F_clk / divisor — round-off error must be < ~3%.

### SPI (Serial Peripheral Interface)
Synchronous, full-duplex, master-slave. 4 wires: SCLK, MOSI, MISO, CS (active-low).

**4 modes** defined by CPOL (clock polarity) and CPHA (clock phase):

| Mode | CPOL | CPHA | Idle clock | Sample on |
|------|------|------|-----------|-----------|
| 0 | 0 | 0 | Low | Rising edge |
| 1 | 0 | 1 | Low | Falling edge |
| 2 | 1 | 0 | High | Falling edge |
| 3 | 1 | 1 | High | Rising edge |

Check the slave device datasheet for its supported mode(s).

### I2C (Inter-Integrated Circuit)
Two wires: SDA (data) and SCL (clock). Open-drain — both pulled up externally.
Multi-master capable. Addresses are 7-bit (or 10-bit extended).

**Transaction sequence**:
```
START → [7-bit addr + R/W] → ACK → [DATA bytes] → ACK/NACK → STOP
```
- **ACK** (SDA pulled low by receiver): "I received the byte, send more"
- **NACK** (SDA stays high): "stop", "error", or "I'm busy"
- **Clock stretching**: slave holds SCL low to pause the master

### PWM (Pulse Width Modulation)
Generated by a hardware timer. Key parameters:
- **Frequency** = 1 / period = F_timer / (prescaler × period_count)
- **Duty cycle** = compare_value / period_count × 100%

Uses: LED dimming, motor speed control, servo position, DAC approximation.

### ADC (Analog-to-Digital Converter)
Converts an analog voltage to a digital number.
- **Resolution**: N bits → $2^N$ levels, LSB = Vref / $2^N$
- **Sample rate**: must obey Nyquist: fs > 2 × f_max_signal
- **Anti-aliasing filter**: low-pass filter before ADC input to remove frequencies > fs/2
- **Reference voltage**: affects accuracy — use a stable, low-noise reference

### CAN Bus
Differential serial bus (CAN_H, CAN_L). Up to 1 Mbit/s at short distances.
Multi-master: any node can transmit when the bus is idle.

**Arbitration** (non-destructive, bit-by-bit):
- Dominant bit (0): CAN_H > CAN_L — wins arbitration
- Recessive bit (1): both lines float to 2.5 V — loses if any node drives dominant
- Lower ID number → more dominant bits → wins arbitration

**Standard frame**: `[SOF | 11-bit ID | RTR | DLC | 0–8 bytes data | CRC | ACK | EOF]`

**Bit stuffing**: after 5 consecutive same-polarity bits, the transmitter inserts one
opposite-polarity bit (receiver strips it).

## Common Interview Questions

1. **Draw the SPI waveforms for Mode 0 and Mode 3.**
   > Mode 0: CLK idle low, data sampled on rising edge. Mode 3: CLK idle high, data
   > sampled on rising edge. Difference: CLK polarity. Both use the same clock edge for
   > sampling but idle differently.

2. **What does a NACK mean during an I2C write?**
   > Could mean: wrong address (no device at that address), device not ready (busy),
   > or device indicating an error after receiving a data byte (e.g., invalid register).

3. **What is clock stretching in I2C?**
   > The slave holds SCL low to pause the master while it processes. The master must
   > wait until SCL is released before continuing. Not all masters support it.

4. **Calculate: UART at 115200 baud, 16 MHz clock. What is the error?**
   > Divisor = 16,000,000 / 115,200 ≈ 138.9. Round to 139.
   > Actual baud = 16,000,000 / 139 = 115,108. Error = (115,200 − 115,108)/115,200 ≈ 0.08%. Fine.

5. **What is bit stuffing in CAN?**
   > After 5 consecutive bits of the same polarity, the transmitter inserts one bit of
   > the opposite polarity. The receiver strips it. This ensures transitions for clock
   > synchronization.

6. **Why does CAN use differential signaling?**
   > CAN_H and CAN_L carry the same signal inverted. A receiver subtracts them, so
   > common-mode noise (affecting both lines equally) cancels out. High noise immunity
   > for automotive/industrial environments.

7. **How do you increase ADC resolution beyond the hardware limit?**
   > **Oversampling**: take 4^N extra samples and sum them; right-shift by N bits to
   > gain N extra bits of resolution. Requires the signal to have enough noise to
   > dither the LSB.

8. **What is an open-drain GPIO output? When do you use it?**
   > Open-drain can only pull the line low (not high). High state relies on an external
   > pull-up resistor. Used for I2C (allows multiple devices to share one line for
   > wired-AND), or for level-shifting between 3.3 V and 5 V logic.
